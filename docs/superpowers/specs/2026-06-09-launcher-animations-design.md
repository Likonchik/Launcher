# Переделка анимаций лаунчера (Slint) — дизайн

Дата: 2026-06-09
Файл реализации: `launcher-slint/ui/app.slint` (~1657 строк)
Подход: **B** — система анимаций + переходы экранов (без рискованного рефакторинга на отдельные файлы)

## Контекст

Лаунчер уже написан на Slint и уже содержит ~83 инлайновых `animate {}`. Проблемы текущих анимаций:

- Экраны переключаются грубо: `visible` + `opacity` без блоков `states`/`in`/`out` — нет осмысленных переходов входа/выхода.
- Используются только базовые easing (`ease-out`, `ease-in-out`) — нет премиального замедления и нет упругости.
- Списки (новости, моды) появляются разом, без каскада.
- Нажатия кнопок и иконки лишены «живого» отклика.

Важное ограничение Slint: **нет общего `scale`-трансформа** для `Rectangle`. Поэтому «пружинный отскок» делается через overshoot-кривую `ease-out-back` на смещении/тени, а «поп» иконки — через анимацию размера `Image` внутри контейнера фиксированного размера.

Внешние библиотеки анимаций не подключаются — Slint-анимации живут в `.slint`, движок встроенный. Никаких новых зависимостей в `Cargo.toml`.

Цель ощущения: **гибрид** — плавная премиальная база для экранов/панелей + точечная упругость на кнопках/иконках.

## Секция 1 — Единый язык движения (motion tokens)

Slint не поддерживает глобальные константы длительностей, поэтому это **соглашение**, применяемое единообразно по всему `app.slint`:

| Токен | Длительность | Easing | Применение |
|-------|-------------|--------|-----------|
| **fast** | 140 мс | `ease-out-quad` | hover, цвет, мелкая реакция |
| **medium** | 280 мс | `ease-out-quint` | панели, тени, прогресс |
| **smooth** | 420 мс | `ease-in-out-quint` | переходы экранов, opacity |
| **spring** | 340 мс | `ease-out-back` | нажатие кнопок, «поп» иконок |

`ease-out-quint` — премиальное мягкое замедление (база гибрида). `ease-out-back` — overshoot, упругость точечно. Сила overshoot — **средняя/ощутимая** (по выбору пользователя).

Существующие `animate` приводятся к этим токенам: каждая анимация попадает в одну из четырёх категорий и получает её длительность+easing.

## Секция 2 — Переходы экранов (`states` + `in`/`out`)

Заменить грубое `visible: !is-authenticated` + `opacity` на блоки `states` с переходами.

**Логин → Главный экран:**
- Карточка логина уходит: `opacity → 0` + `y` уезжает вверх ~16px (`smooth`).
- Главный экран появляется: `opacity 0→1` + `y` подъезжает снизу ~20px (`smooth`, `delay: 80ms`, чтобы выход логина не накладывался встык).

**Настройки / панель загрузки (overlay-панели):**
- Появление: `opacity 0→1` + лёгкий сдвиг/рост снизу (`medium`).
- Скрытие: обратный путь (`medium`).

Шаблон:
```slint
states [
    authed when root.is-authenticated: {
        main.opacity: 1.0;   main.y: 0px;
        login.opacity: 0.0;  login.y: -16px;
        in  { animate main.opacity, main.y { duration: 420ms; delay: 80ms; easing: ease-in-out-quint; } }
        out { animate login.opacity, login.y { duration: 420ms; easing: ease-in-out-quint; } }
    }
]
```

## Секция 3 — Каскадные появления списков (stagger)

Списки новостей и модов выводятся через `for item[idx] in model`. Добавить к анимации `opacity` + `y` задержку по индексу:

```slint
for item[idx] in model: Rectangle {
    opacity: shown ? 1.0 : 0.0;
    y: shown ? 0px : 12px;
    animate opacity, y { duration: 300ms; delay: idx * 50ms; easing: ease-out-quint; }
}
```

Элементы «доезжают» снизу по очереди при первом показе экрана.

## Секция 4 — Кнопки, иконки, прогресс

- **Кнопки** (Gold/Dark/Play/Mini/Nav): hover — `fast`; нажатие — `spring` (overshoot на смещении `y` + тени через `ease-out-back`, сила средняя).
- **Иконки**: «поп» при наведении/нажатии — анимация размера `Image` внутри контейнера фиксированного размера (`spring`).
- **Прогресс загрузки**: ширина полоски и проценты — `medium` `ease-out-quint`, чтобы значение плавно догоняло, а не прыгало.

## Проверка

После правок: `cargo build` в `launcher-slint`, затем запуск лаунчера и визуальная проверка анимаций через скриншот-метод на дисплее `:1` (slint-viewer/wmctrl/ffmpeg x11grab — см. память `slint-widget-screenshot-method`). Убедиться, что `.slint` компилируется без ошибок и переходы/каскады/spring работают.

## Вне рамок (YAGNI)

- Рефакторинг `app.slint` на отдельные файлы по экранам (Подход C).
- Новые зависимости Rust.
- Изменение бизнес-логики, layout'а или контента — только движение.

## Заметки по easing (валидные имена Slint 1.16)

`linear`, `ease(-in|-out|-in-out)`, и Penner-набор с суффиксами `-quad/-cubic/-quart/-quint/-expo/-sine/-circ/-back/-elastic/-bounce`, плюс `cubic-bezier(a,b,c,d)`. Используемые: `ease-out-quad`, `ease-out-quint`, `ease-in-out-quint`, `ease-out-back`.
