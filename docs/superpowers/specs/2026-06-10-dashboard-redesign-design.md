# Редизайн Dashboard — дизайн-спека

Дата: 2026-06-10. Согласовано с владельцем проекта (визуальные варианты выбраны через brainstorm-мокапы в `.superpowers/brainstorm/`).

## Цель

Полный редизайн админ-панели (`dashboard/`, Next.js 15 + React 19): новая визуальная система, новый каркас навигации, современная UX-механика, разбиение монолитных компонентов. Backend API не меняется.

## Решения, принятые на брейншторме

| Вопрос | Решение |
|---|---|
| Объём | Полный редизайн с нуля, выкатка одним заходом |
| Стиль | «Чистая консоль» (Linear/Vercel) + glassmorphism-поверхности |
| Палитра | Тёмный монохром, белый акцент, без зелёного бренд-цвета |
| Навигация | Сайдбар-рейка 64px, раскрывается **по наведению** (не кнопкой) поверх контента + топбар |
| UX-механика | Тосты, confirm-модалки, скелетоны, глобальный поиск (Ctrl+K), live-обновления через SSE |
| Стили | Tailwind CSS 4 (без shadcn/ui) |

## 1. Визуальная система

- Фон приложения: `#070709`.
- Поверхности: стекло — `rgba(255,255,255,0.03–0.05)` + `backdrop-blur(12–20px)`, границы `rgba(255,255,255,0.08)`, радиусы 8–12px.
- Акцент: белый `#fafafa` (активный пункт меню — белая плашка с тёмным текстом, primary-кнопка — белая).
- Семантические цвета (только для статусов): красный `#f87171` — баны/ошибки/опасные действия; янтарный `#fbbf24` — детекты/ожидание; приглушённый зелёный `#a1e3b1` — индикатор «online» (не бренд-цвет).
- Текст: основной `#fafafa`, вторичный `#a1a1aa`, подписи `#8b8b94`, muted `#5f5f66`.
- Шрифт: Inter. Иконки: lucide-react (Material Symbols удаляется).
- Анимации: framer-motion — переходы страниц (fade+y), раскрытие сайдбара, модалки/тосты (AnimatePresence). Сдержанные, 150–250ms.
- Токены объявляются в `@theme` Tailwind 4 в `globals.css`; прежние 886 строк CSS и ~81 инлайн-`style={{…}}` удаляются.

## 2. Каркас (shell)

- **Sidebar** (`components/shell/sidebar.tsx`): рейка 64px (иконки + тултипы недоступны, т.к. hover раскрывает); при наведении мыши плавно расширяется до 240px **оверлеем поверх контента** (контент не сдвигается), фон контента слегка затемняется. Внутри: лого, 5 пунктов (Обзор, Профили, Пользователи, Новости, Античит) с подписями в раскрытом виде, внизу — «Выйти». Активный пункт — белая плашка (motion layoutId).
- **Topbar** (`components/shell/topbar.tsx`): заголовок текущего раздела, кнопка поиска «Ctrl K», живой статус API (зелёная/красная точка, polling /health), аватар-инициал.
- **CommandPalette** (`components/shell/command-palette.tsx`): открывается Ctrl+K/кликом; источники: статические переходы по разделам + поиск пользователей по нику через admin API; стрелки/Enter, Esc.
- **AppFrame** (`components/shell/app-frame.tsx`): прежняя логика guard'а (токен в localStorage, не-админ → /login) + монтирует Sidebar/Topbar/ToastProvider.
- Мобильная адаптация: < 820px — рейка скрывается, в топбаре появляется бургер, открывающий сайдбар оверлеем; таблицы скроллятся горизонтально. (Приоритет — десктоп; мобильная версия функциональна, но без полировки.)

## 3. Архитектура кода

```
dashboard/app/
  globals.css                 — @import "tailwindcss" + @theme токены (короткий)
  layout.tsx                  — html/body, шрифт, AppFrame
  page.tsx                    — Обзор
  login/page.tsx
  profiles/page.tsx, users/page.tsx, news/page.tsx, anticheat/page.tsx — тонкие обёртки
  lib/api.ts                  — единый API-клиент: api<T>(path, opts), токен, обработка
                                ошибок типизированным ApiError, 401 → redirect /login.
                                Заменяет auth.ts::api и две локальные копии request().
  lib/types.ts                — общие типы: Profile, ProfileFile, User, Detection,
                                HwidBan, CheatSignature, Stats и т.п.
  lib/use-sse.ts              — хук подписки на SSE /api/profiles/events c reconnection
components/ui/                — Button, IconButton, Card, StatCard, Table (+Th/Td),
                                Badge, Input, Select, TextArea, Field (label-обёртка),
                                Tabs, Modal, ConfirmDialog + useConfirm(),
                                Toast + ToastProvider + useToast(), Skeleton, EmptyState,
                                Spinner
components/shell/             — app-frame, sidebar, topbar, command-palette
components/profiles/          — profile-list, profile-editor, client-flow (3 шага:
                                Профиль → Клиент → Manifest), files-panel
components/users/             — users-table, user-detail-panel
components/anticheat/         — detections-tab, bans-tab, signatures-tab, stat-row
```

Принципы: один компонент — один файл < ~250 строк; типы импортируются из `lib/types.ts`; никакие компоненты не определяются внутри других файлов-страниц.

## 4. UX-механика

- **Тосты**: ToastProvider (context + AnimatePresence, стек в правом нижнем углу, авто-скрытие 4с, типы success/error/info). Все мутации (сохранение, бан, удаление, скан) показывают тост результата.
- **ConfirmDialog**: promise-based `useConfirm({title, message, confirmLabel, danger})` → модалка; используется для удаления пользователей/профилей/файлов, банов, опасных операций. `window.confirm` удаляется.
- **Скелетоны**: таблицы и статкарты при первой загрузке; повторные обновления — без скелетонов (мягкая подмена данных).
- **EmptyState**: иконка + текст + действие для пустых списков.
- **SSE**: обзор и список профилей подписаны на `/api/profiles/events`; при событии — тихий refetch.
- **Ошибки**: ApiError → toast с сообщением; потеря сессии (401) → редирект на /login с тостом.

## 5. Страницы

- **Обзор**: 4 статкарты (игроки онлайн, аккаунты, детекты 24ч, статус API), лента последних событий (входы из AuthLog, обновления профилей, детекты). Live через SSE + polling /health.
- **Профили**: карточный список; выбор → редактор (двухколоночная форма) + client-flow тремя шагами с прогрессом, как в текущей логике (scan/prepare/manifest), но в новых компонентах. Функциональность сохраняется 1:1.
- **Пользователи**: таблица (логин, email, роль, флаги бан/2FA/HWID, последний вход) с поиском по нику и фильтрами; клик по строке — боковая панель (детали, логи входов, действия: бан/разбан, роль, удалить с confirm).
- **Античит**: табы Детекты / HWID-баны / Сигнатуры; severity-бейджи, выпуск бана с confirm, форма сигнатуры.
- **Новости**: заглушка «в разработке» в новой системе.
- **Login**: центрированная стеклянная карточка: логин, пароль, TOTP (если требуется), состояния ошибок.

## 6. Зависимости

- Добавить: `tailwindcss@4`, `@tailwindcss/postcss`, `lucide-react` (уже есть в package.json).
- Удалить использование Material Symbols (link в layout).
- framer-motion остаётся.

## 7. Критерии приёмки

1. `npm run build` — без ошибок.
2. Живой прогон против локального backend (chrome-devtools): логин (с неверным паролем — тост ошибки), обзор с данными, создание/редактирование профиля, бан/разбан пользователя с confirm-модалкой и тостом, табы античита, Ctrl+K (переход и поиск игрока), сворачивание/раскрытие сайдбара наведением.
3. Никаких обращений к удалённым CSS-классам; grep на `window.confirm`, `material-symbols`, `style={{` в страницах — пусто (точечные исключения допустимы в ui-примитивах).
4. Прежняя функциональность сохранена: все API-вызовы текущих страниц присутствуют в новых компонентах.

## Вне скоупа

- Изменения backend API.
- httpOnly-cookie авторизация (отдельная задача).
- Светлая тема.
- Полировка мобильной версии.
